## Overview

The Game API provides endpoints for retrieving game information, joining games, submitting flags, and managing game participation.

## Public Endpoints

### Get Recent Games

Retrieve games from the last three weeks.

#### GET /api/game/recent

<ParamField query="limit" type="integer" default="0">
  Maximum number of games to return (0-50, 0 = all)
</ParamField>

<ResponseExample>
```json Response
[
  {
    "id": 1,
    "title": "Spring CTF 2026",
    "summary": "Annual spring competition",
    "poster": "/assets/posters/abc123.jpg",
    "startTimeUtc": "2026-03-15T08:00:00Z",
    "endTimeUtc": "2026-03-17T20:00:00Z",
    "participantCount": 156
  }
]
```
</ResponseExample>

<Note>
  This endpoint supports ETag caching. Include `If-None-Match` header for efficient polling.
</Note>

## Participation

### Join Game

Join a game with a team.

#### POST /api/game/{id}

<Note>Requires User authentication and team membership.</Note>

<ParamField path="id" type="integer" required>
  Game ID
</ParamField>

<ParamField body="teamId" type="integer" required>
  ID of the team to join with
</ParamField>

<ParamField body="divisionId" type="integer">
  Division ID (required if game has joinable divisions)
</ParamField>

<ParamField body="inviteCode" type="string">
  Invitation code (if required by game or division)
</ParamField>

<ResponseExample>
```json Request
{
  "teamId": 42,
  "divisionId": 1,
  "inviteCode": "SECRET123"
}
```
</ResponseExample>

<Expandable title="Response Codes">
  - `200` - Successfully joined game
  - `400` - Invalid request (e.g., already in another team, team full, wrong division)
  - `403` - Forbidden (e.g., not team member, invalid invite code)
  - `404` - Game or team not found
</Expandable>

### Leave Game

Leave a game participation.

#### DELETE /api/game/{id}

<Note>Requires User authentication. Can only leave if participation is pending or rejected.</Note>

<ParamField path="id" type="integer" required>
  Game ID
</ParamField>

### Submit Flag

Submit a flag for a challenge.

#### POST /api/game/{id}/challenges/{challengeId}

<Note>Rate limited. Requires User authentication and game participation.</Note>

<ParamField path="id" type="integer" required>
  Game ID
</ParamField>

<ParamField path="challengeId" type="integer" required>
  Challenge ID
</ParamField>

<ParamField body="flag" type="string" required>
  Flag answer (may require encryption, max 127 characters)
</ParamField>

<ResponseExample>
```json Request
{
  "flag": "flag{example_flag_here}"
}
```

```json Response
12345
```
</ResponseExample>

<Info>
  The response is the submission ID. Use the status endpoint to check if the flag is correct.
</Info>

### Check Submission Status

Query the status of a flag submission.

#### GET /api/game/{id}/challenges/{challengeId}/status/{submitId}

<ParamField path="id" type="integer" required>
  Game ID
</ParamField>

<ParamField path="challengeId" type="integer" required>
  Challenge ID
</ParamField>

<ParamField path="submitId" type="integer" required>
  Submission ID from flag submission
</ParamField>

<ResponseField name="status" type="string">
  Submission result: `FlagSubmitted`, `Accepted`, `WrongAnswer`, etc.
</ResponseField>

<ResponseExample>
```json Response
"Accepted"
```
</ResponseExample>
